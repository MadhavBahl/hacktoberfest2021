What is a version control?

Version Control

  • Version control systems are a category of software tools that help a software team manage changes to source code over time.
  • Version control software keeps track of every modification to the code in a special kind of database.
  • If a mistake is made, developers can turn back the clock and compare earlier versions of the code to help fix the mistake while minimizing disruption to all team members.

What is Git?

Git

GIT is the most widely used VCS and is considered to be the best friend of any developer today.

Little working..

  • Git thinks of its data more like a series of snapshots of a miniature filesystem.
  • With Git, every time you commit, or save the state of your project, Git basically takes a picture of what all your files look like at that moment and stores a reference to that snapshot.
  • To be efficient, if files have not changed, Git doesn‚Äôt store the file again, just a link to the previous identical file it has already stored

Git thinks about its data more like a stream of snapshots.

Some cool terms üòé

  1. Snapshot: It records all your files at a given point of time so that you can look up at them anytime later.
  2. Commit: The act of creating a snapshot is called a commit. One is advised to commit his code whenever there are significant changes made.
  3. Repository: The location or digital storage where all your files are stored
  4. Head: The reference to the most recent commit is called Head
  5. Branches: All commits live on some branch and there can be many branches in a single project repository.

Hacktoberfest community by Vinit Shahdeo

ethics of hacktoberfest

Values
  1. Everyone is welcome
  2. Quantity is fun, quality is key
  3. Short-term action, long-term impact

Examples of low-quality contributions
  1. Pull requests that are automated e.g. scripted opening pull requests to remove whitespace / fix typos / optimize images.
  2. Pull requests that are disruptive e.g. taking someone else's branch/commits and making a pull request.
  3. Pull requests that are regarded by a project maintainer as a hindrance vs. helping.
  4. Something that's clearly an attempt to simply +1 your pull request count for October.\
  5. Last but not least, one pull request to fix a typo is fine, but 5 pull requests to remove a stray whitespace is not.
